🧠 神经网络与 LLM 基础

给非技术背景 PM 的 AI 技术直觉指南 — 不需要数学公式，理解"它大概是怎么工作的"就够了

💡 给 PM 的话
你不需要懂反向传播的数学推导，就像乔布斯不需要懂芯片电路设计一样。但你需要理解：神经网络能做什么、不能做什么、为什么 LLM 会"幻觉"、上下文窗口是什么——这些直接影响产品决策。

一、神经网络：AI 的"大脑"是怎么工作的？

神经网络的设计灵感来自人脑。人脑有约 860 亿个神经元，每个神经元通过突触连接到数千个其他神经元。神经网络用软件模拟了这个结构。

🔑 核心概念：神经元（Neuron）

一个神经元做三件事：

  1. 接收输入（比如一张图片的像素值）
  2. 对每个输入乘以一个"权重"（weight），加起来
  3. 通过"激活函数"决定是否"激活"（输出信号）

成千上万个这样的神经元连接在一起，就形成了一个"网络"。

🏠 类比：房价预测

想象你在预测房价。影响房价的因素有：面积、地段、房龄、楼层。神经网络就像一个"超级回归模型"——它自动学习每个因素的权重。面积 100 平米很重要（+权重高），房龄 30 年可能扣分（−权重），但它比普通回归强大得多，因为它能捕捉非线性关系（比如地段好的房子，面积越大溢价越高，这不是简单的线性关系）。

训练：神经网络是怎么"学习"的？

训练过程就像教小孩认猫：

  1. 给神经网络看一张图，它猜"这是狗"（猜错了）
  2. 告诉它"错了，这是猫"
  3. 神经网络调整内部权重，让下次更可能猜对
  4. 重复几万次，直到准确率足够高

这个过程叫做反向传播（Backpropagation）——误差从输出层"反向"传回每一层，调整权重。你不需要懂数学，只需要理解：训练 = 用大量数据调整权重的过程。

二、深度学习：更深的神经网络

"深度"指的是神经网络的层数多。早期的神经网络只有 2-3 层，深度学习可以有几十甚至上百层。

传统机器学习

需要人工"特征工程"——人类告诉模型用什么特征（比如，预测房价要用面积、地段）

深度学习

自动学习特征——模型自己发现"啊，这些像素组合起来是猫的耳朵"

大语言模型

在海量文本上训练的超大规模深度学习模型，参数达千亿级别

为什么"深度"有帮助？ 深层网络可以学习层次化的特征：第一层识别边缘 → 第二层识别形状 → 第三层识别物体部件 → 第四层识别整个物体。就像你认人，先看轮廓，再看五官，最后认出是谁。

三、Transformer：现代 AI 的"发动机"

2017 年，Google 发表了论文《Attention is All You Need》，提出了 Transformer 架构。这是 ChatGPT 和所有现代 LLM 的基础。

🔑 核心创新：注意力机制（Attention）

传统模型处理句子是一个词一个词按顺序处理。Transformer 的突破是：可以同时关注句子中的所有词，并理解它们之间的关系。

比如："银行可以存钱，也可以借钱"——"银行"这个词和"存钱"、"借钱"都有关系。注意力机制让模型在处理"银行"时"注意"到这些相关词。

为什么 Transformer 这么重要？

  • 可以并行处理（不像循环神经网络 RNN 要一个词一个词来），训练速度快得多
  • 可以处理很长的文本，理解远距离的词之间的关系
  • 这就是 GPT（Generative Pre-trained Transformer）名字的由来

四、大语言模型（LLM）是怎么工作的？

训练阶段：预测下一个词

LLM 的训练目标极其简单：给定前面的词，预测下一个词最可能是什么。就是这样。通过在海量文本（整个互联网）上做这个任务，模型"压缩"了人类的知识。

💡 重要理解：LLM 本质上是一个超级强大的"下一个词预测器"。它并没有"理解"意思，但它预测下一个词的能力强到看起来像理解了。这就是为什么它会"幻觉"——它只是在预测下一个看起来合理的词，而不关心事实真假。

推理阶段：生成文本

当你给 ChatGPT 发消息时：

  1. 你的输入被"token 化"（切成词片段）
  2. 模型根据前面所有的 token，预测下一个最可能的 token
  3. 把预测的 token 加回去，再预测下一个
  4. 重复这个过程，直到生成完整回答

这就是为什么 LLM 生成是逐词进行的，也是为什么它有时候会"跑偏"——一旦预测方向错了，后面就跟着错了。

五、PM 必须理解的 7 个 LLM 概念

1. 上下文窗口（Context Window）

模型一次能"看到"多少文本。就像你的工作记忆——超出这个范围的信息模型就"看不到"了。GPT-4o 是 128k tokens（约 10 万汉字），Claude 是 200k tokens。产品含义：如果你的应用需要参考很长的文档，要注意上下文窗口限制。

2. Token

文本被切分的基本单位。英文中大约 1 个词 = 1.3 tokens，中文大约 1 个汉字 = 1.5-2 tokens。你按 token 数量付费，所以上下文越长，成本越高。

3. 温度（Temperature）

控制输出的"随机性"。Temperature = 0 表示每次都选最可能的词（输出稳定但可能死板）；Temperature = 1 表示更随机（有创意但可能不靠谱）。产品含义：客服机器人用低温度，创意写作用高温度。

4. 微调（Fine-tuning）

在通用 LLM 的基础上，用你的专有数据继续训练，让它更擅长特定任务。比如，用你们公司的客服对话记录微调模型，让它更懂你们的业务。但不是所有场景都需要微调——提示工程往往够用了。

5. 嵌入（Embedding）

把文字转换成数字向量（一串数字），让计算机能计算文字之间的"相似度"。这是RAG 系统的基础——把文档转换成嵌入存起来，用户提问时把问题也转成嵌入，找最相似的文档片段。

6. 幻觉（Hallucination）

模型"自信地胡说八道"。因为它本质上是预测下一个词，而不是查询事实数据库。产品含义：不能把 LLM 的输出直接给用户当事实，必须有验证机制（比如让它引用来源）。

7. 推理 / 思维链（Chain of Thought）

让模型"一步一步思考"，可以大幅提升复杂问题的准确率。产品含义：如果你的功能需要模型做复杂推理，要在提示词里加"请一步一步思考"或开启"思考模式"（Claude 的思考功能）。
