🛠️ N8n 实战教程

搭建天气工作流与 AI 智能体 — 从零开始，不写代码

📌 前置说明
本教程基于播客中 Jyothi Nookula 的演示。N8n 是一个低代码/无代码的自动化平台，你不需要写代码（实在需要时，可以让 ChatGPT 帮你写 JavaScript 片段）。

注册：前往 n8n.io 注册免费账号（Cloud 版免费试用，足够完成本教程）。

第一部分：搭建固定工作流（无智能）

我们要搭建的工作流：获取洛杉矶天气 → 格式化成消息 → 发送邮件。每一步都是预先定义好的，没有任何"智能"。

⚙️ 工作流特点

  • 步骤固定，全部执行
  • 没有判断能力
  • 适合重复性任务
  • 可预测、可调试

🤖 智能体（对比）

  • 步骤动态，按需执行
  • 有判断能力
  • 适合开放式任务
  • 更灵活但难调试
1

创建新工作流

登录 N8n → 点击"Create Workflow"→ 命名为"天气邮件报告"。你会看到一个空白画布，中间有一个"Start here"触发节点。

2

添加触发节点（Trigger）

点击"Start here"→ 选择触发方式。我们先用手动触发（Manual Trigger），这样每次点击"Execute"就运行一次。实际使用时可以改成"Schedule Trigger"（每天定时运行）。

3

添加 HTTP 请求节点（获取天气）

点击触发节点右侧的 + → 搜索"HTTP Request"→ 选择 GET 方法，填入以下 URL：

https://api.open-meteo.com/v1/forecast?latitude=34.05&longitude=-118.24¤t=temperature_2m&timezone=America/Los_Angeles

这是免费的天气 API（Open-Meteo），不需要 API Key。34.05, -118.24 是洛杉矶的经纬度。

点击"Execute Step"测试，你应该能看到返回的 JSON 数据，包含当前温度。

4

添加代码节点（格式化消息）

HTTP 请求返回的是原始 JSON，不方便阅读。添加一个"Code"节点来格式化：

// 粘贴到 N8n Code 节点（JavaScript）
const data = $input.first().json;
const temp = data.current.temperature_2m;
const message = `洛杉矶今天的温度是 ${temp}°C（当前）`;

return [{
  json: { message }
}];

不会写代码？ 把上面的需求用普通语言描述给 ChatGPT，让它生成 JavaScript 代码，然后粘贴进来就行。Jyothi 在播客里就是这么做的。

5

添加 Gmail 节点（发送邮件）

点击 + → 搜索"Gmail"→ 选择"Send Message"。首次使用需要连接你的 Google 账号（N8n 会引导你授权）。

  • To：填你的邮箱
  • Subject：今日天气报告
  • Message：拖入上一步的 message 变量

点击"Execute Workflow"，你应该会收到一封天气邮件！

🎉 第一部分完成！

你现在有一个固定工作流：每次运行都走完"触发 → 获取天气 → 格式化 → 发邮件"这 4 步。不管你想不想发邮件，它都会发。这就是"没有智能"的工作流。

第二部分：搭建 AI 智能体（有判断力）

现在我们来搭建一个智能体工作流：用户用自然语言提问，智能体自己决定调用哪个工具。

1

创建新工作流，添加聊天触发器

新建工作流 → 命名为"天气 AI 智能体"→ 触发节点选择"Chat Trigger"（不是 Manual Trigger）。这样你可以通过聊天界面跟智能体对话。

2

添加 AI 智能体节点

点击 + → 搜索"AI Agent"→ 添加。N8n 会自动创建一个智能体节点，包含：

  • Chat Model：需要连接一个 LLM（推荐 GPT-4.1 mini 或 Claude Haiku，便宜够用）
  • Memory：让智能体记住对话历史
  • Tools：智能体可以使用的工具（我们接下来添加）
3

连接 LLM 模型

点击智能体节点里的"Chat Model"→ 选择"OpenAI Chat Model"→ 填入你的 OpenAI API Key（需要在 platform.openai.com 注册获取，有免费额度）。

选择模型：gpt-4.1-mini（便宜，响应快）。

4

添加工具：天气查询

在智能体节点的"Tools"部分，点击"Add Tool"→ 选择"HTTP Request"工具：

  • Name：get_weather
  • Description：获取指定城市的当前天气信息。当用户询问天气时一定要使用此工具。
  • Method：GET
  • URL：https://api.open-meteo.com/v1/forecast?latitude={{ $fromAI("latitude") }}&longitude={{ $fromAI("longitude") }}¤t=temperature_2m&timezone=auto

$fromAI() 是 N8n 的智能参数——LLM 会根据用户问题自动填入纬度和经度。

5

添加工具：发送邮件（可选）

再添加一个工具 → 选择"Gmail: Send Message"→ 命名为 send_email，描述为"当用户要求发送邮件时使用此工具"。

6

测试智能体！

点击右下角的聊天图标 → 在聊天框输入：

"洛杉矶今天的天气是什么？"

观察：智能体调用了 get_weather 工具 → 获取天气数据 → 返回温度。它没有调用 Gmail——因为它判断不需要发邮件。

再试一下：

"把天气信息发送到我的邮箱"

观察：这次智能体调用了 Gmail 工具，发送了邮件。

🎉 第二部分完成！

你现在有一个AI 智能体：它根据用户的自然语言输入，自己判断该用哪个工具。你没有写任何"如果...就..."的逻辑，全是智能体自主决定的。这就是工作流和智能体的本质区别。

三、从实战中提炼的关键洞察

💡 洞察 1：工具描述（Description）极其重要
智能体根据工具的 Description 决定"要不要用这个工具"。如果描述不清晰，智能体可能用错工具，或者不用工具。这叫做"工具提示工程"——跟提示工程一样重要，但很多人忽略。
💡 洞察 2：智能体不是"更聪明"，而是"更有判断力"
智能体不会让天气查询变得更准确。它只是在"要不要查天气"和"要不要发邮件"之间做判断。如果这个判断对你有价值，用智能体；如果步骤固定，用工作流。
💡 洞察 3：先搭工作流，再加智能体
Jyothi 在播客里先搭工作流再搭智能体，这是对的。先理解"没有智能"的版本是怎么工作的，再加上智能体，你才能真正理解两者的区别。
